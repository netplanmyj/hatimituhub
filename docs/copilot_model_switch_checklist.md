# ✅ VS Code Copilot モデル切り替えチェックリスト

## 1. デフォルト運用
- [ ] **普段のコード補完** → **GPT-4.1**  
  - 細かいスニペット、型推論、短い関数はこっちで十分  
- [ ] **設計・要件・リファクタリング** → **Sonnet 4**  
  - 文脈や自然言語理解が必要なときに切り替える  

---

## 2. チャット時のモデル選択
- [ ] チャットパネルを開いたら **右上のモデルピッカー** を確認  
- [ ] **「とりあえず聞く」** → GPT-4.1  
- [ ] **「考えをまとめたい／説明が欲しい」** → Sonnet 4  
- [ ] 迷ったら **両方に同じ質問を投げて比較**

---

## 3. コードレビュー／PR 作成
- [ ] PR の説明文・Issue テンプレ → **Sonnet 4**  
- [ ] コード差分の要約や整理 → **Sonnet 4**  
- [ ] 生成された文章を人間が軽く手直し  

---

## 4. デバッグ／バグ調査
- [ ] バグの修正コード → **GPT-4.1**  
- [ ] バグの原因推定や根本的な解説 → **Sonnet 4**

---

## 5. ライブラリや API の利用
- [ ] 既知のライブラリ・公式パターン → **GPT-4.1**  
- [ ] 初見の複雑なライブラリ・設計検討 → **Sonnet 4**

---

## 6. 長いコードや設計の扱い
- [ ] 大規模ファイルを読ませて整理 → **Sonnet 4**  
- [ ] 必要な部分を抜き出してコード生成 → **GPT-4.1**

---

## 7. 習慣化のヒント
- [ ] **Ctrl/Cmd + Enter** で補完を試して速い答え → GPT-4.1  
- [ ] **「これは設計か？それとも実装か？」と自問** → 設計なら Sonnet 4  
- [ ] **Copilot Chat に「この内容を別のモデルでも答えて」**と投げて比較  

---

👉 この流れに慣れると、  
- **GPT-4.1 = 手を早く動かす職人**  
- **Sonnet 4 = 相談相手の先輩エンジニア**  
というイメージでスムーズに切り替えられるようになります。
